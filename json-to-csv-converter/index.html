Frequently Asked Questions

What format should my JSON be in?

Your JSON must be an array of objects, like [{"key": "value"}, {"key": "value"}]. All object keys become CSV column headers. Nested objects are automatically flattened — for example {"address": {"city": "London"}} becomes address.city.

Is my data safe?

Absolutely. Your data never leaves your browser. Everything runs client-side using vanilla JavaScript. We do not store, log, cache, or transmit any of your data — not even for analytics.

Which delimiter should I choose?

Comma (CSV) is the standard and works in most spreadsheet apps. Semicolon is preferred when your data contains commas. Tab is ideal for pasting directly into Excel or for data with complex text content.

Is there a file size limit?

No hard limit, but very large datasets (100,000+ rows) may take a few seconds to process. For extremely large files, consider using a command-line tool like jq instead.

Is this tool really free?

Yes, completely free. No registration, no API keys, no usage limits, no hidden charges. If you find it useful, share it with a fellow developer.
